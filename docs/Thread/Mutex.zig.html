Lock may be held only once. If the same thread tries to acquire the same mutex twice, it deadlocks. This type supports static initialization and is at most `@sizeOf(usize)` in size. When an application is built in single threaded release mode, all the functions are no-ops. In single threaded debug mode, there is deadlock detection. Example usage: var m = Mutex{}; const lock = m.acquire(); defer lock.release(); ... critical code Non-blocking: if (m.tryAcquire) |lock| { defer lock.release(); // ... critical section } else { // ... lock not acquired } src
